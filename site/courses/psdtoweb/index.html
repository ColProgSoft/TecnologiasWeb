De .PSD A Web Dinámica

Este es un proceso sencillo de como pasar un archivo .psd a una página web dinámica utilizando las tecnologías web HTML CSS JavaScript, entre otras tecnologías web.

Configurar el entorno de desarrollo

Para realizar este ejercicio necesitaremos las siguientes herramientas.

  • Instalar Google Chrome Browser (Pues este por defecto viene con CDT (Crome Developers Tools).)
  • Sublime Text 3 (o cualquier editor de Código)

    • Instrucciones

    Ir a la página de SublimeText3

  • Instalación y configuración de Git
    (Sistema de control de versiones.)

    Instrucciones

    Ir a la página de Git y descargar su version de sistema operativo. en este caso usando linux usaremos el siguiente comando

    $ sudo apt-get install git

  • GitHubCrear una cuenta en GitHub.com

    Instrucciones

    Lo siguiente es vincular nuestra máquina de trabajo con git

    $ git config --global user.mail < email@gmail.com >

    $ git config --global user.name < NombreDeUsuario >

    Ahora lo que tenemos que hacer es crear un repositorio para nuestro proyecto.

    Despues de haber creado el repositorio y la página de GitHub, lo siguiente que debemos hacer es clonarlo en nuestra máquina para trabajar localmente y posteriormente subirlo.

    creamos un directorio para alojar nuestro proyecto

    $ mkdir ProyectosGitHub

    ingresamos en el directorio

    $ cd ProyectosGitHub

    Ahora clonamos nuestro repositorio con el suguiente comando

    $ git clone https://github.com/ColProgSoft/TecnologiasWeb.git

    listamos nuestros repositorios clonados.

    $ ls
    TecnologiasWeb

    Ingresamos en el repositorio, listamos el contenido

    $ cd TecnologiasWeb
    $ ls
    README.md

    Verificamos el Estado del repositorio.

    $ git status
    On branch master
    Your branch is up-to-date with 'origin/master'.
    nothing to commit, working directory clean

    Para cambiar de Branch a las páginas de GitHub, simplemente usamos el siguiente comando:

    $ git checkout gh-pages
    Branch gh-pages set up to track remote branch gh-pages from origin.
    Switched to a new branch 'gh-pages'

    Verificamos el Estado del repositorio.

    $ git status
    On branch gh-pages
    Your branch is up-to-date with 'origin/gh-pages'.
    nothing to commit, working directory clean

    listamos el contenido.

    $ ls
    index.html params.json stylesheets

    creamos un directorio para alojar nuestro sitio

    $ mkdir site

    ingresamos en el directorio

    $ cd site

    Alojamos nuestro sitio dentro de este directorio y listamos los archivos

    $ ls
    css img index.html README.md

    Verificamos el Estado del repositorio.

    $ git status
    On branch gh-pages
    Your branch is up-to-date with 'origin/gh-pages'.
    Untracked files:
    (use "git add < file >..." to include in what will be commited)

    ./

    nothing added to commit but untracked files present (use "git add" to track)

    Lo cual nos muestra un subimos un directorio

    $ cd ..

    Verificamos el Estado del repositorio.

    $ git status
    On branch gh-pages
    Your branch is up-to-date with 'origin/gh-pages'.
    Untracked files:
    (use "git add < file >..." to include in what will be commited)

    site/

    nothing added to commit but untracked files present (use "git add" to track)

    Ahora enviamos los cambios

    $ git add .

    Elige si existe un único directorio.

    Verificamos el Estado del repositorio.

    $ git status
    On branch gh-pages
    Your branch is up-to-date with 'origin/gh-pages'.
    Changes to be commited:
    (use "git add < file >..." to include in what will be commited)

    new file: site/README.md
    new file: site/css/estilos.css
    new file: site/index.html


    Ahora enviamos un commit

    $ git commit -m "Página de inicio."
    3 files changed, 178 insertions(+)
    create mode 100644 site/README.md
    create mode 100644 site/css/estilos.css
    create mode 100644 site/index.html

    Verificamoe el Status

    $ git status
    On branch gh-pages
    Your branch is ahead of 'origin/gh-pages' by 1 commit.
    (use "git push" to publish your local commits)
    nothing to commit, working directory clean

    Ya tenemos nuestro contenido listo para subir a GitHub. para esto usaremos el siguiente comando, el cual por seguridad, nos solicitará el nombre del usuario y la contraseña del mismo.

    $ git push
    Counting objects: 6, done.
    Delta compression using up to 4 threads.
    Compressing objects: 100% (4/4), done.
    Writing objects: 100% (4/4), 2.27 KiB | 0 bytes/s, done.
    Total 4 (delta 1), reused 0 (delta 0)
    To https://github.com/ColProgSoft/TecnologiasWeb.git
    2eaafc7..ca62866 gh-pages -> gh-pages

    Ahora vamos a ver como enviar cambios desde nuestra terminal usando git Vamos a realizar alguna modificación en nuestro archivo index.html para ver lo que sucede. despues usamos el comando:

    $ git status

    el cual nos muestra que se ha realizado modificaciones en los archivos que hayan sido alterados, y para enviar estos cambios usaremos el siguiente comando

    $ git add site/index.html

    indicando la ruta del archivo que se va a modificar, si verificamos con git status nos dirá que se ha modificado el archivo que hayamos modificado.
    lo siguiente, es enviar un mensaje commit usando

    $ git commit -m "Pequeña edición del contenido del sitio."

    Lo último es enviar las modificaciones

    $ git push

    Esto enviará en cuestion de segundos nuestros cambios a los servidores de GitHub para visualizarlos en internet. Ahora podemos

    Felicidades, si llegaste hasta aquí siguiendo todos los pasos, estas listo para trabajar con GitHub.com tu página de GitHub.io ya está en línea y puedes trabajar de manera local y enviar cambios en cualquier momento

    Listo!!!
    Ahora tenemos listo el repositorio en nuestra máquina para trabajar con GitHub de forma local para subir los cambios a realizados a nuestros repositorios, nos ubicamos en la carpeta del proyecto clonado.

    $ cd ProyectoClonado

    creamos los archivos necesarios que necesitamos subir y agregamos las modificaciones que necesitemos en nuestros archivos y ejecutamoe este comando.

    $ git status

    verifica el estado de los paquetes.

    $ git add < file >

    este comando es para elegir a cual será enviado.

    $ git add .

    Elige si existe un único directorio.

    $ git commit -m "Página de inicio"

    Envía un mensaje.

    $ git status

    verifica el estado de los paquetes.

    $ git push

    Este comando subirá los cambios a nuestro sitio.

    Realizamos algunas modificaciones en nuestro sitio, guardamos y ahora verificamos el estado en la terminal.

    $ git pull origin master

    Cuando estemos listos para subir la información al repositorio ejecutamos el siguiente comando en nuestra terminal.

    $ git push origin master

    Este comando subirá los cambios a nuestro sitio.
