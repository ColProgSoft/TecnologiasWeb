De Psd a Web Dinámica.

Este es un proceso sencillo de como pasar un archivo .psd a una página web dinámica utilizando las tecnologías web HTML CSS JavaScript, entre otras

Configurar el entorno de desarrollo

Para realizar este ejercicio necesitaremos las siguientes herramientas.

  • Instalar Google Chrome Browser (Pues este por defecto viene con CDT (Crome Developers Tools).)
  • Sublime Text 3 (o cualquier editor de Código)

    • Instrucciones

    Ir a la página de SublimeText3

  • Instalación y configuración de Git
    (Sistema de control de versiones.)

    Instrucciones

    Ir a la página de Git y descargar su version de sistema operativo. en este caso usando linux usaremos el siguiente comando

    $ sudo apt-get install git

  • GitHubCrear una cuenta en GitHub.com

    Instrucciones

    Lo siguiente es vincular nuestra máquina de trabajo con git

    $ git config --global user.mail < email@gmail.com >

    $ git config --global user.name < NombreDeUsuario >

    Ahora lo que tenemos que hacer es crear un repositorio para nuestro proyecto.

    Despues de haber creado el repositorio y la página de GitHub, lo siguiente que debemos hacer es clonarlo en nuestra máquina para trabajar localmente y posteriormente subirlo.

    creamos un directorio para alojar nuestro proyecto

    $ mkdir ProyectosGitHub

    ingresamos en el directorio

    $ cd ProyectosGitHub

    Ahora clonamos nuestro repositorio con el suguiente comando

    $ git clone https://github.com/ColProgSoft/TecnologiasWeb.git

    listamos nuestros repositorios clonados.

    $ ls
    TecnologiasWeb

    Ingresamos en el repositorio, listamos el contenido

    $ cd TecnologiasWeb
    $ ls
    README.md

    Verificamos el Estado del repositorio.

    $ git status
    On branch master
    Your branch is up-to-date with 'origin/master'.
    nothing to commit, working directory clean

    Para cambiar de Branch a las páginas de GitHub, simplemente usamos el siguiente comando:

    $ git checkout gh-pages
    Branch gh-pages set up to track remote branch gh-pages from origin.
    Switched to a new branch 'gh-pages'

    Verificamos el Estado del repositorio.

    $ git status
    On branch master
    Your branch is up-to-date with 'origin/master'.
    nothing to commit, working directory clean

    Listo!!!
    Ahora tenemos listo el repositorio en nuestra máquina para trabajar con GitHub de forma local para subir los cambios a realizados a nuestros repositorios, nos ubicamos en la carpeta del proyecto clonado.

    $ cd ProyectoClonado

    creamos los archivos necesarios que necesitamos subir y agregamos las modificaciones que necesitemos en nuestros archivos y ejecutamoe este comando.

    $ git status

    verifica el estado de los paquetes.

    $ git add < file >

    este comando es para elegir a cual será enviado.

    $ git add .

    Elige si existe un único directorio.

    $ git commit -m "Página de inicio"

    Envía un mensaje.

    $ git status

    verifica el estado de los paquetes.

    $ git push

    Este comando subirá los cambios a nuestro sitio.

    Realizamos algunas modificaciones en nuestro sitio, guardamos y ahora verificamos el estado en la terminal.

    $ git pull origin master

    Cuando estemos listos para subir la información al repositorio ejecutamos el siguiente comando en nuestra terminal.

    $ git push origin master

    Este comando subirá los cambios a nuestro sitio.
